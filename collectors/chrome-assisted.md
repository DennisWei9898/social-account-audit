# Chrome 輔助採集 — 讀你「自己已登入」的帳號（自查最完整的路）

> **一句話**：健檢**你自己的帳號**時，最完整的做法就是——**你自己登入打開** IG／FB／Threads，讓 **Claude-in-Chrome 唯讀讀你打開的畫面**。這樣連免登入拿不到的**帳號層數字**（粉絲數、Insights 觸及／儲存／reach）都有，因為那些只有登入本人才看得到。做法參考 Dennis 的 `/social-post`（登入牆一律使用者手動登、AI 不自動化登入）。

> **這條 vs 免登入採集怎麼選：**
> - **查你自己的帳號** → **用這條**（最完整，含只有你看得到的 Insights）。
> - **看 role model（別人的公開帳號）、或你不想開瀏覽器套件** → 用免登入的 `threads.md` / `ig.md` / `fb.md`（反正別人的 Insights 你本來也看不到）。
>
> 只要守住下面的護欄（**你自己登入、AI 不代登、人工節奏不暴力翻**），風險就落在低區——真正會封號的是「AI 自動登入 ＋ 暴力翻大量」，不是「讀你自己打開的頁面」。

---

## 先搞懂：這跟「會被封號的那種」差在哪

我們的安全紅線案例（R9）是：有人用「AI **自動登入**本人 IG ＋ 暴力翻 90 天貼文生報告」→ 被平台判定機器人、**帳號停用**。

**真正該禁的是「自動登入」＋「暴力大量翻取」這兩件事，不是「用 Claude-in-Chrome」本身。**

| | ❌ 會封號的做法（R9） | ⚠️ 這條路徑（可控 opt-in） |
|---|---|---|
| 登入 | AI **自動登入**（觸發登入流程/輸入帳密） | **你自己先手動登入**，AI 完全不碰登入 |
| 動作 | 程式化**暴力翻大量貼文**（90 天） | **唯讀、人工節奏、少量**查看幾個後台頁 |
| 讀/寫 | 混在一起 | **只讀，絕不在同流程發文/DM/改設定** |
| 結果 | 帳號停用 | 風險大幅降低（但非零，見下） |

## 硬護欄（缺一不可，AI 與你都要守）

1. **你自己先手動登入**目標平台（IG／FB）。**AI 絕不自動登入、絕不輸入帳密、絕不觸發登入流程**；一遇到登入牆就停下、請你手動登入後才繼續。
2. **只讀不寫**：AI 只 `navigate` + 讀畫面上的數字，**不 click／不 type 去改任何東西**。只看後台數字頁（帳號粉絲數、單篇貼文 Insights 的觸及／儲存／分享）。
3. **人工節奏、少量**：一次看幾個頁面就好。**禁止程式化連翻幾十上百則貼文**——那正是 R9 被封的主因。要大量歷史數據？用平台官方「下載你的資訊／匯出」功能自己匯出。
4. **讀寫永久分離**（拆 Lethal Trifecta）：這個「查數字」流程裡**絕不發文、絕不私訊、絕不改隱私或帳號設定**。要發文請走另一個獨立流程（如 `/social-post`），不要跟查詢混在一起。
5. **風險自負**：即使完全照做，「在已登入的 Meta 帳號上做任何自動化」仍有**非零**的偵測風險。不能接受這個風險，就退回純手動匯出／截圖——那是零帳號風險的做法。

## 怎麼用（給 Claude 的指令樣板）

> 「我已經在 Chrome 自己登入好我的 IG 了。請用 Claude-in-Chrome **只唯讀**查我 @我的帳號 的**粉絲數**，以及最近 5 篇貼文的 **Insights（觸及、儲存、分享）**。規則：不要點任何按鈕、不要輸入任何東西、不要發文或私訊、遇到要重新登入就停下來讓我登。只把讀到的數字整理成表格給我。」

- 前置：`mcp__Claude_in_Chrome__*` 工具可用（沒有就停，不要用別的方式硬做、不要模擬數字）。
- 平台權限：read 級 navigate 通常自動過；本路徑**不需要**任何 write 級授權（因為只讀）——若 AI 要求 click/type 授權，代表它想做寫入動作，**拒絕並停下**。

## 拿得到什麼（免登入拿不到、這條補上的）

- **帳號層**：真實粉絲數、追蹤數、貼文總數。
- **單篇 Insights**（僅你自己的貼文可見）：觸及（reach）、儲存（saves）、分享、來自主頁/探索的分佈等——這些是**只有登入本人後台才看得到**的數字，免登入永遠拿不到。

## 退場條件（看到就立刻停）

- 平台跳出「異常活動／請驗證你是本人」→ **立刻停止所有自動化**，改人工。
- 任何需要輸入密碼／二階段驗證的畫面 → 停，交給你手動。
- 你只是想健檢、不是想冒帳號風險——**猶豫就退回手動匯出**，帳號比數字值錢。
