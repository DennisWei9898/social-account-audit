# social-account-audit · 社群帳號健檢（loop-engineering 版）

把一個「不知道哪裡卡、數字上不去」的社群帳號，體檢成一份可執行的路線：定位、受眾、平台、30 天內容日曆、0–3–6 個月變現時間表。**這是給你健檢「你自己」的帳號用的**——結論因人而異，不是套公版。

## 一句話：這個 skill 怎麼運作
```
指揮官（規劃/審查/拍板）
  → 派研究 AI 並行三路（受眾 / 同業整體 / 平台）— 每條附來源
  → maker 產出 01–06
  ⇄ verifier（換一個乾淨 context 的 AI）驗硬條件
  → 接 social-cards-engine 生 30 天圖卡
```

## 觸發詞
健檢我的帳號 · IG 健檢 · Threads 健檢 · 幫我看我的社群帳號 · 我的帳號定位怎麼抓 · 我想靠社群變現先幫我健檢 · 幫我盤點我的 IG/Threads

---

## 🧱 Loop engineering 骨架（這個 skill 的靈魂，不可省）
1. **指揮官只做判斷**：規劃、審查、拍板留給最強的腦（你，或一個強模型）；查資料、比對、產草稿這種吞吐量大的粗活，派給便宜又快的 AI。「一個人，開一間迷你 AI 公司。」
2. **研究讀寫分離**：研究 AI 只查證、不下結論——回報「事實＋來源」，改計畫是指揮官的事。每條發現附來源 URL，查不到就說查不到，**禁止腦補**。
3. **maker / verifier 分離**：產出（maker）與驗收（verifier）用**不同 context 的 AI**。verifier 預設有罪推定（「maker 說 done 是主張不是證明」），**只驗客觀硬條件**（字元數、欄位齊全、來源可回查、有無編造），逐條 PASS/FAIL＋證據。🚫 verifier 不准以「風格/觀點不合我意」打回——那是使用者的資產。
4. **三條紅線**（貫穿全程）：
   - **反虛構**：不替使用者編造成就、不掰精確數字（要用數字先查證或改可辯護的概括）。健檢建在假前提上，全部白做。
   - **安全路線**：永不用瀏覽器自動登入本人帳號抓資料（見下）。
   - **寫審分離**：產出與驗收分開；視覺產出逐張肉眼看。

## 🛡️ 開跑前安全鐵則（必讀）
**明文禁止**用任何瀏覽器自動化登入使用者本人的帳號去抓後台資料。**已有實測案例**：用「AI 自動登入本人 IG 翻貼文生報告」被平台判定為機器人操作、帳號遭停用。這是 Lethal Trifecta（真帳號＋自動化＝被判機器人）。

**安全的拿資料方式（Step 1 只准這幾種）**：
1. 使用者用平台內建 Insights **手動匯出**（截圖或匯出檔）。
2. 使用者直接**貼截圖**（貼文、後台數據）。
3. 公開頁面的**唯讀**讀取（免登入、無憑證）——只讀不寫。

一句話：**讀公開頁 OK，登入本人帳號自動操作 = 禁止。**

## 流程步驟（照順序跑）

### Step 0 · 訪談鎖目標
先搞清楚使用者「想幹嘛」（變現／漲粉／接案／導流本業），目標不同結論完全不同。請使用者回答：目標與成功樣貌、真實資產與身分、想打的受眾、帳號現況。**要求誠實、禁誇大**（見反虛構紅線）。

### Step 1 · 讀現況（走安全路線）
請使用者手動匯出/貼截圖近 12–15 則貼文的主題+互動、bio、表現最好與最差的貼文。整理成「帳號現況」摘要。

### Step 2 · 派 AI 並行研究（fan-out）
指揮官派**三路**研究，每路只查證、附來源、查不到就說：
- **受眾**：使用者想打的人在乎什麼、痛點、為什麼付錢（找付費意願證據）。
- **同業整體**：這個領域**整體**怎麼做、什麼格式/主題有效、常見變現方式。**看整片森林、歸納通則——不點名、不盯單一帳號逐字抄**（太針對、也學不到全貌）。
- **平台**：適合使用者目標的引流/變現平台（含費用/門檻，數字附來源、無來源標「推估」）。

### Step 3 · 找定位縫隙
三路交叉比對，找「沒人好好佔、又剛好是使用者強項」的定位縫隙。**品味：市場縫隙 ≠ 使用者本人身分**——別把市場空缺硬套成使用者不是的樣子；定位要建在真實資產上。

### Step 4 · 產出 01–06（maker）
一次領一份，別一次全要。填進 `templates/`：
- **01 定位包**：一句話定位 + bio（≤150 字元，含「做什麼＋給誰＋單一 CTA 動詞結尾」）+ 內容三支柱。
- **02 Personas**：≥2 個付費 persona，各附「付費意願證據＋可點來源 URL」。
- **03 平台決策**：適合的平台與理由（數字附來源）。
- **04 30 天內容日曆**：正好 30 則，六欄（日期/平台/格式/主題/hook/CTA＋漏斗階段），主題與日期不重複，每列標【認知/興趣/轉換】。
- **05 變現時間表**：0–3–6 個月里程碑，成果指標**含具體數字**（別喊空話）。
- **06 可重跑流程**：把這次健檢沉澱成使用者自己能重跑的 SOP。
文案禁 AI 八股腔（不要「在當今數位時代」「讓我們一起」這類套語）；留言鉤（教學型貼文）＝末頁燒「留言『關鍵字』我把[具體零成本資源]傳你」→ 私訊導使用者的私域，把互動變名單。

### Step 5 · 獨立 verifier 驗收（換乾淨 context）
開一個新 context 的 AI 當 verifier，逐條驗（附證據，FAIL 退回重做，不放水）：
- bio 字元數 ≤150、三要素齊全
- 每 persona 有付費證據＋可點來源 URL
- 30 天日曆正好 30 列、六欄齊全、主題/日期不重複、每列有漏斗標籤
- 變現里程碑每列成果指標含數字
- 有無 AI 八股腔 / 有無編造成就或掰數字
- 視覺產出（若有圖卡）**逐張肉眼看**，機器檢查只當篩選器

### Step 6 · 接 social-cards-engine 生 30 天圖卡
把 01 的風格方向 + 04 的日曆丟進姊妹專案 **social-cards-engine**（github.com/DennisWei9898/social-cards-engine），找出品牌風格、把 30 則主題生成 IG 圖卡＋文案。

## 安裝方式（Claude Code）
把這個 `skill/` 資料夾複製或連結到你的 skills 目錄：
```
cp -r skill ~/.claude/skills/social-account-audit
# 或： ln -s "$(pwd)/skill" ~/.claude/skills/social-account-audit
```
然後在 Claude Code 說「健檢我的帳號」即可觸發。沒有 Claude Code 也行——直接照 `../prompts/` 的 Prompt 一步步貼給 claude.ai 對話。

## 語氣守則（產文案時）
繁體中文、口語、給乾貨、反賣課姿態；禁 AI 八股腔；數字/主張引用研究一律標來源。

## 搭配
- `social-cards-engine`：Step 6 把日曆變 IG 圖卡。
- 若有 `fable-commander` / maker-verifier 類 skill：可把整套健檢當一個正式的指揮官 loop 跑。
