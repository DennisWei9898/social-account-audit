# Case Study：用作者本人公開帳號 @now6pm（Threads）驗證健檢採集流程

> **這份文件在證明什麼？**
> 社群帳號健檢流程要可信，第一步不是「相信 AI 說的數字」，而是**證明採集本身可被任何人複驗**。
> 本 case 用作者本人的公開 Threads 帳號 **@now6pm** 當白老鼠，全程**唯讀、免登入、無憑證**跑一次真實採集，把每一格數字標上信心等級、附上可重跑的 `curl` 指令，並誠實揭露哪些欄位拿不到。
>
> **範圍限定：只用 Threads，不放 IG。** 這是刻意決定——避免把底層本名帳號一起曝光。profile 免登入時雖偵測得到一條 outbound 外部連結，但本文**不揭露其目標**。
>
> **安全鐵則（寫進文件也自己遵守）**：全程唯讀、免登入、無任何憑證；請求間隔 ≥ 3 秒、不暴力重試；**禁止**自動化登入本人帳號；第三方 proxy（r.jina.ai）回傳一律視為不可信、需人工覆核。

## 3. ★ 留言鉤反差對比列（同一帳號的 A/B 對照）

這是本 case 的王牌證據：**同一個作者、同一週內、同樣是文字敘事**，只差在「有沒有明確 CTA 要求留言關鍵字」，互動結構就完全倒掛。

| 對照組 | 貼文 | 讚 | 留言 | **留言/讚 比** | 貼文設計 |
|---|---|---|---|---|---|
| ★ 留言鉤（置頂） | DaeNAbik6qk | 32 | **166** | **5.19×** | 結尾明確 CTA：「留言『健檢』，我立馬發給你」 |
| 一般敘事貼文 | DadJq3CE2ia | 7 | 4 | **0.57×** | 純敘事，無留言 CTA |

**異常點在哪？**
- 正常貼文（#2）留言/讚 = 0.57 < 1 —— 這符合直覺：多數人只按讚、少數才留言。
- 置頂貼文（#1）留言/讚 = **5.19 > 1** —— **留言數是讚的 5 倍**，這在自然互動下幾乎不會發生。唯一合理解釋是「留言誘因」設計：把資源（開源專案）鎖在「留言關鍵字才發你」的動作後面，硬把讚的能量導去留言。
- 這不是跟別人比，而是**同一帳號自我對照**——排除了粉絲數、題材、發文時間、演算法波動等雜訊，直接把「留言鉤」這一個變因獨立出來。

**旁證（同帳號其他貼文也重現同型態）：**
- #7 DaaXewzE41A：讚 1 / 留言 **5**，留言/讚 = **5×**，CTA 同樣是「留言『指揮官』我私訊給你」→ 留言鉤型態重現。
- #4 Daah-FhEyPF：讚 20 / **分享 23**，分享/讚 > 1，屬另一種「可存檔工具型」內容驅動的**分享鉤**。

> 結論（僅就本帳號實爬數據）：在 @now6pm 上，「留言誘因 CTA」與「留言 > 讚」的倒掛現象高度共現。這為健檢報告裡「留言鉤有效」的假設提供了**同源、可複驗的 meta-proof**，而不是空口宣稱。

## 5. 方法限制誠實揭露區

### 哪些欄位是實爬到的真數據
- 帳號顯示名稱、handle、**followers（231）**、Bio 全文、Bio 標籤 —— profile 直接渲染。
- 每則貼文的：相對時間（16h/1d/2d）、永久連結（post ID）、內文全文、非零互動數、附圖 CDN URL、外連貼文的 GitHub OG 卡標題、置頂標記（Pinned）。
- 置頂與一般貼文的互動數 —— 且與 2026-07-07 基準爬**跨兩日一致**（#1、#2 標高信心）。

### 哪些欄位因登入牆 / JS / 私密拿不到（誠實標「抓不到／退回手動」）
- **觀看數（views）**：整份輸出無任何 view 數字 → 抓不到，不估算。
- **貼文總數、追蹤中（following）**：profile 未渲染 → 抓不到。
- **互動數為 0 的欄位**：不渲染 → 少於 4 個數字時欄位歸屬需推論（#5、#6、#7 標中信心）。
- **留言內容與留言者名單**：需登入 → 抓不到。
- **精確發文時間戳**：只有相對時間（16h/1d/2d），需自行換算絕對時間。
- **歷史貼文**：免登入僅可見最新約 7 則，之後撞登入牆（輸出結尾出現 `Log in to see more from now6pm.`）→ 完整時間線退回手動。

> **不誇大聲明**：本流程**不保證**任何欄位「一定抓得到」。Threads 前端改版、快取延遲、rate limit 都可能讓某次採集少欄位或失敗。凡拿不到就標「抓不到／退回手動」，不以推估填補。

## 缺口與限制

- **觀看數 / views**：輸出中完全不存在 → 抓不到，禁止估算。
- **貼文總數、following 數**：profile 未渲染 → 退回手動。
- **0 值互動欄位**：不渲染，導致 #5/#6/#7 欄位歸屬需依動作列順序推論（標中信心）。
- **留言內容、留言者、私訊**：需登入 → 抓不到。
- **精確發文時間**：僅相對時間，需換算。
- **歷史貼文**：登入牆截斷在最新約 7 則，時間線不完整。
- **數字時效**：免登入頁為 Meta 快取，可能與即時值有分鐘級落差；複驗時以當下為準。

## 風險

- **第三方 proxy 不可信**：r.jina.ai 回傳內容可能被改寫或注入，一律**視為不可信、需人工覆核**（依 CLAUDE.md / MEMORY 鐵則）。
- **前端改版脆弱性**：Threads DOM 結構隨時可能改，數字位置解析會 break，需定期回歸測試。
- **Rate limit / 封 IP**：過度頻繁請求可能被限流 → 請求間隔 **≥ 3 秒、不暴力重試**。
- **ToS 邊界**：僅採**公開、免登入**資料，**不繞登入牆、不自動化登入本人或他人帳號**；讀（採集）與寫（發文）永久分離，以拆解 Lethal Trifecta。
- **隱私範圍**：本 case 刻意只用 Threads、不揭露 profile 上偵測到的外部連結目標，避免曝光底層本名帳號。
