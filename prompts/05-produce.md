# Prompt · ⑤ 產出 01–06 + 驗收

> 研究（RESEARCH.md）出來後，貼給 Claude，一份一份產。產完務必用「驗收 Prompt」叫另一個乾淨 context 的 AI 把關。

## 產出 Prompt（一次領一份，別一次全要）

```
根據我的訪談目標與 RESEARCH.md（貼上），產出【01 定位包】：
- 一句話定位（記得住）
- IG bio 全文，≤150 字元，含「做什麼＋給誰＋單一 CTA（動詞結尾）」三要素
- 內容三支柱（每支柱佔比 + 範例主題 3 個），對齊我鎖定的定位方向
文案禁 AI 八股腔（不要「在當今數位時代」「讓我們一起」這類套語），用真人口語。
```

同樣的方式，一份一份產：
- **02 Personas**：≥2 個付費 persona，每個附「付費意願證據＋具體可點來源 URL」。
- **03 平台決策**：適合我目標的平台選擇與理由，數字附來源。
- **04 30 天內容日曆**：正好 30 則，每則六欄（日期/平台/格式/主題/hook/CTA＋漏斗階段），主題與日期都不重複，每列 CTA 標【認知/興趣/轉換】其一。
- **05 變現時間表**：0–3–6 個月里程碑，每個成果指標**含具體數字**（別喊空話）。
- **06 可重跑流程**：把這次健檢寫成我自己能重跑的 SOP。

## 留言鉤（把互動變名單，教學型貼文用）
在 04 日曆的教學型貼文，最後一頁/結尾燒一句留言鉤：
> 留言「<好記關鍵字>」，我把 <具體、零成本的資源> 私訊給你。
把「核心資源」鎖在留言後才給（飢餓式）→ 衝留言訊號、餵演算法二次推播、順便收私域名單。

## 驗收 Prompt（換一個乾淨 context 的 AI，寫審分離）

```
你是獨立驗收員，預設有罪推定，禁當啦啦隊。逐條驗下面硬條件，附證據：
1. bio 字元數 ≤150（實際數出來）
2. bio 三要素（做什麼/給誰/單一 CTA 動詞結尾）齊全
3. 每個 persona 有付費意願證據 + 可點來源 URL
4. 30 天日曆正好 30 列、六欄齊全、主題/日期不重複、每列有漏斗標籤
5. 變現里程碑每列成果指標含數字
6. 文案有沒有 AI 八股腔 / 有沒有編造成就或掰數字
逐條 PASS/FAIL + 證據；FAIL 的退回重做，不要放水。
```
